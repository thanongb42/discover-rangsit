1. แนวทางหลักในการพัฒนา Mobile App จาก Web Project นี้
────────────────────────────────────────────────────────────────────────────────

โปรเจคนี้เป็น PHP MVC Web Application มีฐานข้อมูล MySQL และ REST API อยู่บางส่วน
แนวทางที่เหมาะสมที่สุดคือ:

  [A] พัฒนา Mobile App แยก → ให้ App ยิง API มาที่ Backend PHP เดิม
      - ข้อดี: ใช้ Backend และ Database เดิมได้เลย ไม่ต้องเริ่มใหม่
      - ข้อดี: ข้อมูลซิงก์กันระหว่าง Web และ App โดยอัตโนมัติ
      - ข้อเสีย: ต้องขยาย API ให้ครอบคลุม (ปัจจุบันมี /api/places, /api/search ฯลฯ)

  [B] แปลง Web เป็น PWA (Progressive Web App)
      - แปลง Web ให้ติดตั้งได้บนมือถือ มี offline mode และ push notification
      - ข้อดี: ไม่ต้องขึ้น App Store, ใช้โค้ดเดิม 90%
      - ข้อเสีย: ฟีเจอร์บางอย่างจำกัด เช่น การเข้าถึง GPS, Camera ไม่ลื่นเท่า Native

  แนะนำ: ทำ [A] เพราะ User Experience ดีกว่า และรองรับ GPS/Camera ได้เต็มรูปแบบ

2. ภาษาและ Framework ที่ควรใช้ (แยกตามค่าย)
────────────────────────────────────────────────────────────────────────────────

  ┌─────────────────────────────────────────────────────────────────────┐
  │  CROSS-PLATFORM (แนะนำสำหรับทีมเล็ก — เขียนครั้งเดียว ได้ทั้งสองค่าย) │
  └─────────────────────────────────────────────────────────────────────┘

  [1] Flutter (by Google)
      - ภาษา: Dart
      - IDE: Android Studio หรือ VS Code + Flutter Extension
      - จุดเด่น: UI สวยงาม, Performance ดีใกล้เคียง Native
      - เหมาะกับ: โปรเจคนี้มาก เพราะมี Map, รูปภาพ, API call
      - เรียนรู้: ง่ายถึงปานกลาง
      - Community: ใหญ่มาก มี Package เยอะ

  [2] React Native (by Meta/Facebook)
      - ภาษา: JavaScript / TypeScript
      - IDE: VS Code
      - จุดเด่น: ถ้าทีมรู้ React อยู่แล้ว เรียนรู้เพิ่มน้อย
      - จุดด้อย: Performance ต่ำกว่า Flutter เล็กน้อย
      - เหมาะกับ: ทีมที่มีประสบการณ์ JS/React

  ┌─────────────────────────────────────────────────────────────────────┐
  │  NATIVE (Performance สูงสุด แต่ต้องเขียนแยก Android และ iOS)        │
  └─────────────────────────────────────────────────────────────────────┘

  [3] Android Native
      - ภาษา: Kotlin (แนะนำ) หรือ Java
      - IDE: Android Studio (ฟรี)
      - SDK: Android SDK (ฟรี)
      - จุดเด่น: เข้าถึง Feature เครื่อง Android ได้ 100%

  [4] iOS Native
      - ภาษา: Swift (แนะนำ) หรือ Objective-C
      - IDE: Xcode (ฟรี — ต้องใช้ macOS เท่านั้น!)
      - จุดเสีย: ต้องมี Mac จึงจะพัฒนาและ Build ได้

  >>> สรุป: แนะนำ Flutter สำหรับโปรเจคนี้ เพราะทีมเล็ก ต้องการ Map และ API <<<

3. สิ่งที่ต้องเตรียม/พัฒนาเพิ่มใน Backend (PHP) ก่อน
────────────────────────────────────────────────────────────────────────────────

  ปัจจุบัน API ที่มีอยู่:
    GET  /api/places         - ดึงรายการสถานที่
    GET  /api/search         - ค้นหา
    GET  /api/categories     - ดึง category
    GET  /api/top-places     - Top places
    POST /api/place/review   - เพิ่มรีวิว
    POST /api/place/like     - Like
    GET  /api/weather        - สภาพอากาศ
    GET  /api/air-quality    - คุณภาพอากาศ

  API ที่ต้องพัฒนาเพิ่มสำหรับ Mobile:
    POST /api/auth/login          - Login ด้วย Email/Password (return JWT Token)
    POST /api/auth/register       - สมัครสมาชิก
    POST /api/auth/google         - Google OAuth สำหรับ Mobile
    POST /api/auth/line           - LINE Login สำหรับ Mobile
    GET  /api/auth/me             - ดึงข้อมูล Profile ตัวเอง
    GET  /api/place/{slug}        - รายละเอียดสถานที่
    POST /api/place/create        - สร้างสถานที่ใหม่ (multipart/form-data)
    GET  /api/user/businesses     - ธุรกิจของฉัน
    GET  /api/user/reviews        - รีวิวของฉัน
    POST /api/user/profile/update - อัปเดต Profile

  ระบบ Authentication สำหรับ Mobile ควรใช้:
    - JWT (JSON Web Token) แทน Session Cookie
    - เพราะ Mobile App ไม่ใช้ Cookie แบบ Browser

4. ข้อจำกัดของแต่ละค่าย (Android vs iOS)
────────────────────────────────────────────────────────────────────────────────

  ANDROID
  -------
  - ข้อดี: ไม่ต้องจ่าย Review Fee รายปี (จ่ายครั้งเดียว $25)
  - ข้อดี: Publish ได้เร็ว Review ใช้เวลา 1-3 วัน (บางครั้งไม่กี่ชั่วโมง)
  - ข้อดี: ไม่ต้องใช้ Mac ในการพัฒนา
  - ข้อเสีย: Device หลากหลายมาก ต้องทดสอบหลาย Screen size
  - ข้อเสีย: มัลแวร์/App ปลอมเยอะกว่า iOS (แต่ไม่กระทบการพัฒนา)

  iOS (Apple)
  -----------
  - ข้อดี: User คุณภาพสูง, ใช้จ่ายต่อหัวสูงกว่า Android
  - ข้อดี: Device มีจำนวนแบบน้อยกว่า ทดสอบง่ายกว่า
  - ข้อเสีย: ต้องใช้ macOS + Xcode ในการ Build และ Submit เท่านั้น
  - ข้อเสีย: Apple Review เข้มงวดมาก Review นาน 1-7 วัน
  - ข้อเสีย: ต้องจ่าย Apple Developer Program $99/ปี
  - ข้อเสีย: Apple บังคับ Privacy Policy ต้องชัดเจน และ App ต้องมีประโยชน์จริง
  - ข้อเสีย: ถ้าใช้ In-App Purchase ต้องจ่าย Commission 15-30% ให้ Apple

5. ค่าใช้จ่ายในการนำ App ขึ้น Store
────────────────────────────────────────────────────────────────────────────────

  GOOGLE PLAY STORE (Android)
  ---------------------------
  - Google Play Developer Account: $25 (จ่ายครั้งเดียวตลอดชีพ)
  - ค่า Commission จาก In-App Purchase: 15% (รายได้แรก $1M/ปี) / 30% (ส่วนที่เกิน)
  - ค่า Subscription App: 15% (ถ้า User อยู่เกิน 12 เดือน)
  - ไม่มีค่า Review หรือค่าบำรุงรายปี

  APPLE APP STORE (iOS)
  ---------------------
  - Apple Developer Program: $99/ปี (หรือ 3,490 บาท/ปีโดยประมาณ)
    (ต่ออายุทุกปี ถ้าหมดอายุ App จะถูกลบออกจาก Store)
  - หน่วยงานรัฐ/NGO: สมัคร Apple Developer Enterprise Program $299/ปี
    (สำหรับ distribute ภายในองค์กร ไม่ต้องผ่าน App Store)
  - ค่า Commission จาก In-App Purchase: 15-30%
  - TestFlight (Beta Testing): ฟรี

  ค่าใช้จ่ายอื่นๆ ที่ต้องคำนึง
  ----------------------------
  - Google Maps SDK สำหรับ Mobile: มี Free Tier $200/เดือน (ปกติพอใช้)
    ถ้า API Call เยอะ อาจต้องจ่ายเพิ่ม
  - Firebase (Push Notification, Analytics): มี Free Tier เพียงพอสำหรับ App ขนาดเล็ก-กลาง
  - Apple Push Notification Service (APNs): ฟรี (รวมอยู่ใน Developer Program แล้ว)
  - Server/Hosting: ใช้ Server เดิมได้เลย ไม่มีค่าใช้จ่ายเพิ่ม

6. Roadmap แนะนำ (ลำดับขั้นตอน)
────────────────────────────────────────────────────────────────────────────────

  Phase 1: เตรียม Backend API (1-2 เดือน)
  - เพิ่ม JWT Authentication ใน PHP
  - ขยาย API endpoints ให้ครบ
  - ทดสอบ API ด้วย Postman

  Phase 2: พัฒนา Mobile App (2-4 เดือน)
  - ติดตั้ง Flutter SDK
  - สร้าง UI: Home, Map, Place Detail, Search, Profile
  - เชื่อมต่อ API
  - ทดสอบบน Android Emulator และ iOS Simulator

  Phase 3: ทดสอบและ Deploy (1 เดือน)
  - Beta Test ด้วย TestFlight (iOS) และ Internal Testing (Android)
  - แก้ Bug
  - สร้าง Developer Account และ Submit

  Phase 4: หลัง Launch
  - ติดตาม Crash Report ด้วย Firebase Crashlytics (ฟรี)
  - อัปเดต App ตาม Feedback

7. สรุปข้อแนะนำสำหรับโปรเจค Discover Rangsit โดยเฉพาะ
────────────────────────────────────────────────────────────────────────────────

  Framework ที่เหมาะสมที่สุด: Flutter
  ภาษา: Dart
  IDE: VS Code หรือ Android Studio
  Map: Flutter Map (Leaflet บน Flutter) หรือ Google Maps Flutter Plugin
  Auth: JWT + Google Sign-In Flutter Plugin
  HTTP Client: Dio (Flutter Package)
  State Management: Provider หรือ Riverpod

  ค่าใช้จ่ายเริ่มต้นที่ต้องจ่ายก่อน Launch:
  - Google Play: $25 (ครั้งเดียว)
  - Apple Developer: $99/ปี (~3,500 บาท/ปี)
  - รวม: ~3,800 บาท ในปีแรก

  สำหรับ App ภาครัฐ/เทศบาล:
  - สมัคร Apple Developer Program ในนามองค์กร (ราคาเดียวกัน $99/ปี)
  - ใช้ Google Play Free of Charge หากเป็น App รัฐบาล (ตรวจสอบเงื่อนไขเพิ่มเติม)
